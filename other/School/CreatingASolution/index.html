iv. CHANGES MADE TO DESIGN / CODE EXPLANATION

Page Navigation

I made some changes to the page navigation style. First, I decided that having a fixed headbar was not efficient because it took up a lot of space for the page and may not be useful for the user all the time. Second, I decided that the flip boxes will turn into buttons which will redirect the user to the page they want. This is because there is not enough space to fit the info within the small boxes. It was not an intellegant design choice. Finally, I got rid of the next page for my main page and included it in my information pages because the purpose of the main page is to introduce to the users the sections and categories of my website. I also included a footer because I did not want to cram all the buttons into one section.

External CSS and Javascript

There are many benefits of an external CSS and Javascript file in most web developing situations so I decided to scrap the idea of inline CSS and Javascript (In most files, when appropriate). External CSS and JS is so effective because 1) everything is stored in one file 2) once changed and saved, the changes are applied to all html files that reference to that css or js file 3) pages load quicker once main css file is loaded.

Design (Animations, Color, Responsivness)

I decided upon a design/theme that I have not prepared in my developing ideas section. I choose the color of green because it reflects upon the color of forests, leaves, nature, and trees. My original design was to create a colorful background image for children because it is very colorful, however I tried that and it did not work. It was too distracting and the color palatte did not match up. The page looked messy. So i decided to stay with a simplistic white background with animated leaves falling down, giving the page the feeling as if the user is in the forest trying to learn things. Students will definitely be interested and the design looks very appealing and professional for a kids website. The reponsiveness of pages was also important because I also wanted the page to work on a smaller window or if viewing it from a device, like an ipad or phone because those devices are more accessible to students than computers. I learned about @media and styling certain id and classes under the condition that the screen is at a certain width or height.

My overall experience coding this site was amazing, as I learned a lot about web development through this. I realized that making many design changes are normal and that I have to learn while creating the final solution.
